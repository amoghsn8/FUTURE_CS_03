# Task 3 – API Security Risk Analysis



This repository contains the deliverables for \*\*Task 3 of the Future Interns Cybersecurity Internship\*\*, which involved performing an \*\*API Security Risk Analysis\*\* on publicly accessible demonstration APIs.



The objective of this task was to analyze API endpoints and identify potential security risks related to \*\*authentication, authorization, data exposure, input validation, and request handling\*\*. The testing was performed in a controlled environment using publicly available APIs designed for learning and development purposes.



---



## Tools Used



- Postman  

- Browser Developer Tools  

- Kali Linux Environment  



---



## APIs Tested



The following public APIs were analyzed during this task:



- ** JSONPlaceholder API **  

&#x20; https://jsonplaceholder.typicode.com  



- ** DummyJSON API **  

&#x20; https://dummyjson.com  



- ** ReqRes API **  

&#x20; https://reqres.in  



These APIs provide simulated data for testing API interactions and security analysis.



---



## Analysis Performed



During the testing process, several API security checks were conducted:



- Testing API endpoints for authentication requirements  

- Checking authorization controls by modifying object identifiers  

- Observing API responses for excessive data exposure  

- Testing input validation using modified query parameters  

- Evaluating API behavior when sending repeated requests  



Evidence screenshots were collected during the testing process and are included in this repository.



---



## Disclaimer



The analysis was conducted strictly for \*\*educational purposes\*\* using publicly available demonstration APIs.  

No exploitation attempts were made, and no real systems were targeted during this task.
